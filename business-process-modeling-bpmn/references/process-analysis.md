# Process Analysis

Methods for analyzing and improving business processes.

---

## Value Stream Analysis

### Purpose
Identify value-adding vs non-value-adding activities to eliminate waste.

### Activity Classification
| Type | Description | Action |
|------|-------------|--------|
| Value-Adding (VA) | Customer would pay for | Optimize |
| Business Value-Adding (BVA) | Required but not valued | Minimize |
| Non-Value-Adding (NVA) | Pure waste | Eliminate |

### Seven Types of Waste
1. **Transport**: Moving materials/info unnecessarily
2. **Inventory**: Excess materials/data waiting
3. **Motion**: Unnecessary movement of people
4. **Waiting**: Idle time between activities
5. **Overproduction**: Producing more than needed
6. **Over-processing**: More work than required
7. **Defects**: Errors requiring rework

### Value Stream Mapping
1. Define product/service family
2. Draw current state map
3. Identify waste and improvement areas
4. Design future state map
5. Create improvement plan

## Cycle Time Analysis

### Time Components
| Metric | Definition |
|--------|------------|
| Process Time | Actual work time |
| Wait Time | Time between activities |
| Cycle Time | Total time start to finish |
| Lead Time | Time from request to delivery |
| Takt Time | Rate needed to meet demand |

### Efficiency Calculation
```
Process Efficiency = Process Time / Cycle Time × 100%
```

### Analysis Steps
1. Map all process activities
2. Measure time for each activity
3. Measure wait times between activities
4. Calculate total cycle time
5. Identify longest waits
6. Target wait time reduction

## Bottleneck Analysis

### Identification Methods
- **Queue analysis**: Where work accumulates
- **Utilization analysis**: Resources at 100%
- **Time analysis**: Longest duration activities

### Theory of Constraints
1. **Identify** the constraint
2. **Exploit** maximize constraint output
3. **Subordinate** align everything to constraint
4. **Elevate** increase constraint capacity
5. **Repeat** find new constraint

### Resolution Strategies
| Strategy | When to Use |
|----------|-------------|
| Add capacity | Demand exceeds capability |
| Reduce batch size | Large batches cause delays |
| Eliminate variability | Inconsistent processing |
| Parallel processing | Sequential bottlenecks |
| Load balancing | Uneven distribution |

## Root Cause Analysis

### 5 Whys Technique
1. State the problem
2. Ask "Why?" and answer
3. Repeat 4 more times
4. Identify root cause
5. Develop countermeasure

### Fishbone Diagram Categories
- **People**: Skills, training, staffing
- **Process**: Procedures, workflows
- **Technology**: Systems, tools
- **Materials**: Inputs, data
- **Environment**: External factors
- **Management**: Policies, decisions

### Pareto Analysis
- 80% of problems from 20% of causes
- Prioritize high-impact issues
- Focus resources on vital few

## Process Metrics

### Efficiency Metrics
| Metric | Formula |
|--------|----------|
| Process Efficiency | Process Time / Cycle Time |
| First Pass Yield | (Good Units / Total Units) |
| Throughput | Units / Time Period |
| Capacity Utilization | Actual Output / Maximum Output |

### Quality Metrics
| Metric | Formula |
|--------|----------|
| Defect Rate | Defects / Total Units |
| Rework Rate | Rework Items / Total Items |
| Error Rate | Errors / Transactions |

### Time Metrics
| Metric | Definition |
|--------|------------|
| Average Cycle Time | Mean time per instance |
| On-Time Delivery | % completed by deadline |
| Touch Time | Total actual work time |
