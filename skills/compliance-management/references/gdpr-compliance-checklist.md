# GDPR Compliance Checklist

Essential requirements for GDPR compliance.

## Lawful Basis for Processing

- [ ] Consent
- [ ] Contract
- [ ] Legal obligation
- [ ] Vital interests
- [ ] Public task
- [ ] Legitimate interests

## Data Subject Rights

- [ ] Right to access
- [ ] Right to rectification
- [ ] Right to erasure ("right to be forgotten")
- [ ] Right to restrict processing
- [ ] Right to data portability
- [ ] Right to object
- [ ] Rights related to automated decision-making

## Privacy by Design and Default

- [ ] Data minimization
- [ ] Purpose limitation
- [ ] Storage limitation
- [ ] Pseudonymization and encryption
- [ ] Default privacy settings

## Data Protection Impact Assessment (DPIA)

- [ ] Identify when DPIA required
- [ ] Conduct DPIA for high-risk processing
- [ ] Document findings
- [ ] Implement mitigations
- [ ] Consult supervisory authority if needed

## Data Breach Management

- [ ] Breach detection procedures
- [ ] 72-hour notification to supervisory authority
- [ ] Notification to affected individuals (if high risk)
- [ ] Breach documentation

## Data Protection Officer (DPO)

- [ ] Determine if DPO required
- [ ] Appoint qualified DPO
- [ ] Provide resources and independence
- [ ] Publish contact details

## International Data Transfers

- [ ] Adequacy decisions
- [ ] Standard Contractual Clauses (SCCs)
- [ ] Binding Corporate Rules (BCRs)
- [ ] Derogations for specific situations

## Records of Processing Activities

- [ ] Maintain processing records
- [ ] Document purposes
- [ ] Identify data categories
- [ ] List recipients
- [ ] Note retention periods
- [ ] Describe security measures

## Vendor Management

- [ ] Data Processing Agreements (DPAs)
- [ ] Vendor due diligence
- [ ] Ongoing monitoring
- [ ] Sub-processor management
