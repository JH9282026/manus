### Naming Conventions

**Process Naming:**
- Format: `[Domain]_[Function]_[SubProcess]_[Version]`
- Example: `FIN_AP_InvoiceProcessing_v2.1`
- Use clear, descriptive names
- Avoid abbreviations except standard ones

**Activity Naming:**
- Verb + Object format
- Example: "Review Invoice", "Approve Payment"
- Action-oriented language
- Consistent terminology

**Document Naming:**
- Include process reference
- Date or version indicator
- Document type identifier
- Example: `FIN_AP_SOP_InvoiceApproval_v1.2_20240315`
