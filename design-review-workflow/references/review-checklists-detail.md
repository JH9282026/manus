# Review Checklists Detail

Complete element-by-element checklists for conducting thorough design reviews.

---

## Full Review Checklist

### 1. Visual Design (25%)

#### Color
- [ ] Color harmony and contrast meet WCAG AA
- [ ] Primary color used consistently for key actions
- [ ] Semantic colors applied correctly (success, warning, error, info)
- [ ] Neutral palette provides sufficient range for UI hierarchy
- [ ] Color works for colorblind users (test with simulation)
- [ ] Dark backgrounds have adequate text contrast

#### Typography
- [ ] Body text is 16px+ for readability
- [ ] Heading hierarchy uses 3+ distinct levels
- [ ] Line height is 1.5–1.7 for body text
- [ ] Line length stays within 60–75 characters
- [ ] Font weights create clear hierarchy
- [ ] No more than 2–3 font families used

#### Layout & Alignment
- [ ] All elements snap to the grid
- [ ] Consistent margins and padding throughout
- [ ] Visual hierarchy guides the eye logically
- [ ] Whitespace is used intentionally
- [ ] No orphaned elements or awkward gaps

#### Visual Polish
- [ ] Shadows, borders, and radii are consistent
- [ ] Icons are the same style and weight
- [ ] Images are properly sized and cropped
- [ ] No pixelated or stretched assets
- [ ] Hover and active states are polished

---

### 2. User Experience (25%)

#### Clarity
- [ ] Each screen has a clear primary purpose
- [ ] Primary action is immediately visible
- [ ] Labels and copy are clear and concise
- [ ] Users know what to do without instructions

#### Navigation
- [ ] Navigation structure matches mental model
- [ ] Current location is always visible
- [ ] Back navigation is always available
- [ ] Search is accessible for complex apps

#### Task Completion
- [ ] Critical flows have minimal steps
- [ ] Progress indicators for multi-step tasks
- [ ] Success confirmation for completed actions
- [ ] Undo available for destructive actions

#### Error Prevention & Handling
- [ ] Form validation prevents common errors
- [ ] Error messages are specific and helpful
- [ ] Data is preserved on errors
- [ ] Recovery paths are clear

---

### 3. Consistency (15%)

- [ ] Same component looks identical everywhere
- [ ] Interaction patterns are consistent (e.g., all modals behave the same)
- [ ] Spacing follows the established scale
- [ ] Terminology is consistent throughout
- [ ] Icon style is uniform

---

### 4. Responsiveness (15%)

- [ ] Mobile layout is usable and not just "shrunk desktop"
- [ ] Touch targets are 44px+ on mobile
- [ ] Content priority is maintained on small screens
- [ ] Breakpoint transitions are smooth
- [ ] No horizontal scrolling on mobile
- [ ] Images are responsive

---

### 5. Accessibility (10%)

- [ ] Color contrast meets WCAG AA (4.5:1 for text)
- [ ] Focus states are visible on all interactive elements
- [ ] Touch targets are 44×44px minimum
- [ ] Form fields have associated labels
- [ ] Important info not conveyed by color alone
- [ ] Keyboard navigation order is logical

---

### 6. Brand & Polish (10%)

- [ ] Design aligns with brand guidelines
- [ ] Tone and voice are consistent
- [ ] Design feels professional and trustworthy
- [ ] Memorable or delightful elements present
- [ ] Empty states are designed (not just blank)
- [ ] Loading states are designed
