# Writing Guide

Detailed guidance for writing compelling case study content.

---

## Headlines and Titles

### Headline Formulas
- **Results-led**: "How [Company] Achieved [Metric] with [Product]"
- **Challenge-led**: "[Company] Solves [Problem] with [Product]"
- **Quote-led**: "'[Compelling Quote]' - [Company Name]"
- **Number-led**: "[X]% [Improvement] in [Timeframe]"

### Headline Examples
- "How Acme Corp Reduced Costs by 40% in 6 Months"
- "From Manual Chaos to Automated Efficiency: The TechStart Story"
- "'It Changed Everything' - How GlobalRetail Transformed Operations"

---

## Executive Summary

### Structure
1. Company identification and context
2. Challenge statement
3. Solution overview
4. Key results (2-3 metrics)
5. Customer endorsement quote

### Length
- 100-200 words
- Scannable in 30 seconds
- Standalone value

### Example
```
[Company], a leading [industry] company with [context], 
faced [specific challenge]. After implementing [Product], 
they achieved [result 1] and [result 2] within [timeframe]. 
"[Quote from customer]," says [Name], [Title] at [Company].
```

---

## Challenge Section

### Elements to Include
- Business context (size, industry, situation)
- Specific pain points (concrete examples)
- Impact of the problem (costs, time, quality)
- Previous attempts to solve
- Stakes if problem continued

### Writing Tips
- Use specific details, not generalities
- Include numbers where possible
- Create emotional resonance
- Set up the solution naturally

### Example Transitions
- "The team was spending X hours per week on..."
- "Without a solution, [Company] risked..."
- "Previous attempts had failed because..."

---

## Solution Section

### Elements to Include
- Selection criteria/process
- Why your product was chosen
- Implementation approach
- Key features utilized
- Timeline and milestones

### Feature Presentation
- Don't list features; show them in action
- Connect features to specific problems
- Use customer's language

### Example Structure
```
After evaluating [options], [Company] chose [Product] because 
[reason]. The implementation focused on [approach], starting 
with [initial phase] and expanding to [full rollout] over 
[timeframe].
```

## Results Section

### Result Categories
| Category | Examples |
|----------|----------|
| Quantitative | 40% reduction, $500K saved, 3x faster |
| Qualitative | Better visibility, improved collaboration |
| Strategic | Market expansion, competitive advantage |
| Operational | Streamlined processes, reduced errors |

### Presenting Metrics
- Lead with most impressive result
- Group related metrics
- Use before/after comparison
- Include timeframes
- Explain context for numbers

### Results Format
```
Key Results:
• [X]% improvement in [metric]
• $[X] saved annually
• [X] hours saved per week
```

## Quote Integration

### Quote Placement
- Executive summary: Key endorsement
- Challenge: Problem acknowledgment
- Solution: Why they chose you
- Results: Validation of outcomes
- Closing: Recommendation

### Quote Types
| Type | Example |
|------|----------|
| Results | "The ROI exceeded our expectations..." |
| Process | "Implementation was smoother than..." |
| Recommendation | "I'd recommend to any company facing..." |
| Emotional | "Finally, we could focus on..." |

### Formatting
- Block quote for emphasis
- Full attribution (Name, Title, Company)
- Pull quotes for key statements

---

## Writing Style

### Tone
- Professional but engaging
- Confident without exaggeration
- Customer-focused, not self-promotional
- Specific and concrete

### Do's
- Use active voice
- Include specific details
- Let customer's voice shine
- Focus on business outcomes

### Don'ts
- Overly salesy language
- Vague claims without evidence
- Technical jargon without context
- Excessive superlatives
