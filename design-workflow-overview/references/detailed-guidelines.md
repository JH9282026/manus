# Phase Details - Detailed Reference

Detailed reference content for design workflow overview.

---

## Phase Details
### Phase 1: Research (Foundation)

**Duration**: 10-15% of total effort

**Activities**:
1. Gather project requirements and constraints
2. Research target audience and industry
3. Collect 10-20 inspiration references from Dribbble, web
4. Analyze 3-5 competitor designs
5. Document findings and insights

**Outputs**:
- Research brief
- Inspiration moodboard
- Competitive analysis report
- Design direction recommendations

**Key Skills**:
- `design_research.md`
- `inspiration_gathering.md`
- `competitive_analysis.md`

---

### Phase 2: UX Foundation

**Duration**: 15-20% of total effort

**Activities**:
1. Define user personas (2-3 primary users)
2. Map out user flows for key tasks
3. Structure information architecture
4. Create wireframes for key screens

**Outputs**:
- User persona documents
- User flow diagrams
- Site/app structure map
- Wireframe mockups

**Key Skills**:
- `user_personas.md`
- `user_flows.md`
- `information_architecture.md`
- `wireframing.md`

---

### Phase 3: Visual Design

**Duration**: 30-35% of total effort

**Activities**:
1. Establish color palette (primary, secondary, neutral, semantic)
2. Select typography system (headings, body, accents)
3. Define grid system and layout rules
4. Design component library
5. Source/create imagery and icons
6. Establish spacing scale and rhythm

**Outputs**:
- Color system documentation
- Typography scale
- Grid specifications
- Component library
- Asset collection
- Spacing guidelines

**Key Skills**:
- `color_palettes.md`
- `typography.md`
- `layout_grids.md`
- `component_design.md`
- `imagery_iconography.md`
- `spacing_rhythm.md`

---

### Phase 4: Responsive & Platform Design

**Duration**: 15-20% of total effort

**Activities**:
1. Adapt designs for all breakpoints
2. Apply platform-specific patterns (iOS/Android)
3. Design micro-interactions and animations
4. Ensure touch targets and mobile usability

**Outputs**:
- Responsive mockups (mobile, tablet, desktop)
- Platform-specific variants
- Interaction specifications
- Animation guidelines

**Key Skills**:
- `responsive_design.md`
- `mobile_app_design.md`
- `interaction_design.md`

---

### Phase 5: Review & Iteration

**Duration**: 15-20% of total effort (per iteration cycle)

**Activities**:
1. Conduct systematic design review
2. Score design against rubric (1-10)
3. Identify specific areas for improvement
4. Check accessibility compliance
5. Implement iterations until 9/10+

**Outputs**:
- Design review report
- Score breakdown by category
- Improvement recommendations
- Accessibility audit
- Refined design iterations

**Key Skills**:
- `design_review.md`
- `design_scoring.md`
- `design_iteration.md`
- `accessibility_review.md`

---

### Phase 6: Delivery

**Duration**: 5-10% of total effort

**Activities**:
1. Create comprehensive design documentation
2. Prepare developer handoff assets
3. Export assets in required formats
4. Document components and interactions

**Outputs**:
- Design specification document
- Asset export package
- Component documentation
- Implementation notes

**Key Skills**:
- `design_documentation.md`
- `handoff_preparation.md`

## Iteration Guidelines
### When to Iterate

| Current Score | Action Required |
|---------------|----------------|
| 1-3/10 | Major rework - revisit UX foundation |
| 4-5/10 | Significant revision - address multiple areas |
| 6-7/10 | Focused improvement - target weak areas |
| 8/10 | Polish and refinement |
| 9-10/10 | Ready for delivery |

### Common Iteration Patterns

**Pattern 1: UX-First Iteration**
```
Low usability score → Revise user flows → Update wireframes → Apply visual design
```

**Pattern 2: Visual Refinement**
```
Low aesthetics score → Revisit color/typography → Adjust spacing → Polish components
```

**Pattern 3: Consistency Pass**
```
Inconsistent elements → Audit components → Standardize patterns → Update all instances
```

**Pattern 4: Accessibility Fix**
```
WCAG failures → Check contrast → Fix touch targets → Add focus states → Verify
```

## Quality Assurance Checkpoints
### Checkpoint 1: Post-Research
- [ ] Clear understanding of requirements
- [ ] 10+ quality inspiration references
- [ ] Competitive analysis complete
- [ ] Design direction defined

### Checkpoint 2: Post-UX
- [ ] Personas validated
- [ ] User flows logical and complete
- [ ] Information architecture clear
- [ ] Wireframes approved

### Checkpoint 3: Post-Visual Design
- [ ] Color palette harmonious and accessible
- [ ] Typography system consistent
- [ ] Components designed and documented
- [ ] Visual hierarchy clear

### Checkpoint 4: Post-Responsive
- [ ] All breakpoints addressed
- [ ] Platform conventions followed
- [ ] Interactions defined
- [ ] Touch targets adequate

### Checkpoint 5: Final Review
- [ ] Overall score 9/10+
- [ ] All accessibility checks pass
- [ ] Documentation complete
- [ ] Assets exported correctly

## Tips for Success
1. **Don't skip research** - Inspiration gathering saves time later
2. **Iterate early** - Catch issues in wireframes, not final designs
3. **Score honestly** - Objective evaluation drives improvement
4. **Document as you go** - Don't leave documentation for the end
5. **Embrace feedback loops** - Each iteration improves quality
