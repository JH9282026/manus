
### Understanding Audio Descriptions

Audio descriptions (also called video descriptions or described video) provide narration of visual information for blind and low-vision viewers. A narrator describes important visual elements during natural pauses in dialogue.

### What to Describe

**Essential Visual Information**:
- Actions and movements
- Character appearances (when relevant)
- Scene settings and locations
- On-screen text and graphics
- Facial expressions and body language
- Time and scene changes

**Prioritization**:
1. Information essential to understanding plot
2. Character identification
3. Setting and context
4. Actions and reactions
5. Visual details that enhance experience

### Audio Description Styles

**Standard Audio Description**:
- Fits within natural pauses
- No alteration to original audio
- Most common approach
- Works for most content

**Extended Audio Description**:
- Video pauses to allow description
- Used when pauses are insufficient
- More comprehensive coverage
- May affect viewing experience

**Integrated Description**:
- Descriptions part of original production
- Characters narrate their actions
- Eliminates need for separate track
- Requires planning during production

### Audio Description Workflow

**Production Process**:
1. Analyze video for description opportunities
2. Write description script
3. Time scripts to available pauses
4. Record narration
5. Mix with original audio
6. Quality review
7. Deliver as separate audio track or mixed version

**Description Writing Best Practices**:
- Use present tense
- Be objective, not interpretive
- Describe what you see, not what you think
- Use concise language
- Time descriptions precisely
- Match tone to content

**Narration Recording**:
- Clear, neutral voice
- Consistent volume with original audio
- Professional recording quality
- Appropriate pacing
- Match content tone

### Audio Description Delivery

**Separate Audio Track**:
- SAP (Secondary Audio Program) for broadcast
- Alternate audio track in streaming
- User-selectable
- Platform-dependent implementation

**Mixed Version**:
- Audio description mixed with main audio
- Single audio track
- No viewer control needed
- Used for specific accessible versions

**Standards and Regulations**:
- FCC requirements for broadcast
- CVAA (21st Century Communications and Video Accessibility Act)
- OFCOM requirements (UK)
- Percentage requirements for programming
