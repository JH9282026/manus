# Lean Methodology

Comprehensive guide to Lean principles, tools, and implementation.

---

## Core Lean Principles

### 1. Value

Define value from the customer's perspective:
- What are customers willing to pay for?
- Distinguish value-adding from non-value-adding activities
- Focus resources on creating value

### 2. Value Stream

Map all steps required to deliver value:
- Identify every action in the value stream
- Categorize as value-adding, necessary non-value-adding, or pure waste
- Visualize end-to-end process

### 3. Flow

Create smooth, uninterrupted movement:
- Eliminate batching, queues, and interruptions
- Design for single-piece flow where possible
- Remove obstacles to continuous movement

### 4. Pull

Produce only what is demanded:
- Avoid overproduction and inventory buildup
- Use signals (kanban) to trigger production
- Respond to actual customer demand

### 5. Perfection

Continuously pursue elimination of all waste:
- Strive for zero defects
- 100% value-adding activities
- Improvement is an endless journey

## Value Stream Mapping (VSM)

### Process

1. **Select Product Family**: Choose product/service with similar processing steps
2. **Create Current State Map**: Document all steps, information flows, metrics
3. **Analyze Current State**: Calculate lead time, process time, identify waste
4. **Design Future State Map**: Envision improved flow with waste eliminated
5. **Develop Implementation Plan**: Create action items to achieve future state

### Key Metrics

- **Process Time (PT)**: Time actually working on product
- **Lead Time (LT)**: Total time from start to finish including waiting
- **Value-Added Ratio**: PT / LT x 100%
- **First Pass Yield**: Percentage passing without rework

## 5S Implementation

### Step-by-Step Process

**1. Sort (Seiri)**
- Red tag items of questionable necessity
- Remove unnecessary items
- Keep only what's needed

**2. Set in Order (Seiton)**
- Designate locations for all items
- Label everything clearly
- Make frequently used items easily accessible

**3. Shine (Seiso)**
- Clean workplace thoroughly
- Create cleaning schedules
- Maintain equipment

**4. Standardize (Seiketsu)**
- Create visual controls
- Establish standards for maintaining 5S
- Document best practices

**5. Sustain (Shitsuke)**
- Regular audits
- Leadership commitment
- Continuous reinforcement

## Kaizen Events

### Structure

- **Duration**: 3-5 days
- **Team Size**: 6-12 people
- **Scope**: Specific, bounded problem
- **Authority**: Team empowered to make changes

### Phases

1. **Preparation** (1-2 weeks before): Define scope, gather data, assemble team
2. **Day 1**: Training, observation, waste identification
3. **Days 2-3**: Solution development, testing, implementation
4. **Day 4**: Finalize changes, document standards
5. **Day 5**: Present results, celebrate, plan follow-up

## Kanban Systems

### Principles

- **Visualize Work**: Make all work visible on boards
- **Limit WIP**: Set maximum items at each stage
- **Manage Flow**: Focus on smooth movement
- **Make Policies Explicit**: Define rules for work movement
- **Implement Feedback Loops**: Regular reviews
- **Improve Collaboratively**: Team-based improvement

### Types

- **Production Kanban**: Authorizes production
- **Withdrawal Kanban**: Authorizes movement between processes
- **Supplier Kanban**: Signals external replenishment

---

## Just-In-Time (JIT)

### Elements

- **Takt Time**: Rate to meet customer demand
- **Continuous Flow**: Single-piece or small-batch movement
- **Pull Production**: Upstream produces only what downstream needs
- **Quick Changeover (SMED)**: Rapidly switch between products
- **Level Production (Heijunka)**: Smooth volume and mix

---

## Poka-Yoke (Error-Proofing)

### Categories

- **Prevention**: Makes errors impossible
- **Detection**: Immediately identifies errors
- **Warning**: Alerts before error occurs

### Examples

- Form fields validating input format
- Fixtures accepting only correct orientation
- Sensors detecting missing components
- Checklists preventing skipped steps
