# Visual Design and Slides

Principles and best practices for creating impactful presentation visuals that enhance understanding and retention.

---

## Core Design Principles

### The 6x6 Rule

Limit text to maintain readability and focus:
- Maximum 6 bullet points per slide
- Maximum 6 words per bullet
- Better yet: aim for less
- Use multiple slides rather than cramming content

### One Idea Per Slide

Each slide should communicate a single concept:
- Avoid cognitive overload
- Allow audience to focus
- Make slides scannable
- Use builds/animations to reveal points sequentially if needed

### Visual Hierarchy

Guide audience attention through design:
- **Size** — larger elements draw attention first
- **Color** — contrast highlights key information
- **Position** — top-left to bottom-right reading pattern
- **White space** — emptiness creates focus

## Typography Best Practices

### Font Selection

**Sans-serif fonts for presentations:**
- Arial, Helvetica, Calibri for clarity
- Avoid decorative fonts for body text
- Maximum 2 font families per deck
- Consistent font usage throughout

**Size guidelines:**
- Title: 36-44 pt minimum
- Body text: 24-32 pt minimum
- Never below 18 pt (unreadable from distance)
- Test readability from back of room

**Emphasis techniques:**
- **Bold** for emphasis (not underline)
- *Italics* for quotes or foreign terms
- Color for highlighting
- Avoid ALL CAPS except for short headings

## Color Psychology and Usage

### Color Meanings and Associations

**Blue** — trust, professionalism, stability (most common corporate color)
**Red** — urgency, passion, danger, energy
**Green** — growth, health, environment, money
**Yellow** — optimism, caution, creativity
**Purple** — luxury, creativity, wisdom
**Orange** — enthusiasm, affordability, fun
**Black** — sophistication, power, formality
**White** — simplicity, cleanliness, space

### Color Scheme Guidelines

**Contrast for readability:**
- Dark text on light background (or vice versa)
- Avoid low-contrast combinations (gray on gray)
- Test for colorblind accessibility
- Use tools like WebAIM Contrast Checker

**Consistent palette:**
- 2-3 primary colors maximum
- Use brand colors when applicable
- Create visual consistency across slides
- Avoid rainbow effects

**Strategic color use:**
- Highlight key data points
- Differentiate categories in charts
- Draw attention to calls to action
- Maintain meaning consistency (don't switch color meanings)

## Image Selection and Usage

### High-Quality Visuals

**Image quality standards:**
- Minimum 1920x1080 resolution for full-slide images
- Avoid pixelation or compression artifacts
- Professional photography over clip art
- Authentic images over stock photo clichés

**Image sources:**
- Unsplash, Pexels (free high-quality)
- Adobe Stock, Shutterstock (paid)
- Custom photography when budget allows
- Ensure proper licensing and attribution

### Image Composition

**Full-bleed images:**
- Extend to slide edges for impact
- Overlay text with semi-transparent boxes for readability
- Use rule of thirds for composition
- Ensure text placement doesn't obscure key image elements

**Image and text integration:**
- Sufficient contrast between text and background
- Use drop shadows or glows sparingly
- Consider image cropping to create space for text
- Align text with image composition

## Data Visualization

### Chart Type Selection

| Data Relationship | Best Chart Type | Use Case |
|---|---|---|
| Comparison | Bar chart | Compare values across categories |
| Trend over time | Line chart | Show changes over time periods |
| Part-to-whole | Pie chart | Show percentage breakdown (max 5 slices) |
| Distribution | Histogram | Show frequency distribution |
| Correlation | Scatter plot | Show relationship between variables |
| Ranking | Horizontal bar | Show ordered list of items |

### Chart Design Best Practices

**Simplify for clarity:**
- Remove chart junk (unnecessary gridlines, 3D effects, shadows)
- Direct label data points instead of legends when possible
- Use color strategically to highlight key data
- Round numbers for readability (use 2.5M not 2,547,893)

**Tell a story with data:**
- Add descriptive title that states the insight
- Annotate key data points
- Use callout boxes for context
- Show comparison or benchmark when relevant

**Accessibility:**
- Don't rely solely on color to convey information
- Use patterns or shapes in addition to color
- Provide alt text descriptions
- Ensure sufficient contrast

## Slide Layout Templates

### Title Slide

**Essential elements:**
- Presentation title (clear and compelling)
- Subtitle or tagline (optional)
- Speaker name and credentials
- Date and event/organization
- High-impact background image or brand visual

### Content Slide Layouts

**Text-focused slide:**
- Clear heading
- 3-5 bullet points maximum
- Ample white space
- Optional supporting icon or small image

**Image-focused slide:**
- Full-bleed or large image
- Minimal text overlay
- Clear headline
- Use for emotional impact or visual examples

**Split slide (50/50):**
- Image on one side, text on other
- Useful for before/after, comparison, or explanation
- Maintain visual balance

**Data slide:**
- Chart or graph as focal point
- Descriptive title stating the insight
- Minimal additional text
- Source citation if needed

## Animation and Transitions

### When to Use Animation

**Appropriate uses:**
- Reveal bullet points sequentially to control pacing
- Build complex diagrams step-by-step
- Show process or timeline progression
- Emphasize key data points

**Avoid:**
- Gratuitous effects (spinning, bouncing, dissolving)
- Different animations for each slide (inconsistent)
- Animations that distract from content
- Automatic animations that can't be controlled

### Transition Guidelines

**Consistency:**
- Use same transition throughout (or none)
- Simple fade or cut is professional
- Avoid elaborate transitions (cube rotate, etc.)
- Match transition speed to presentation pace

## Accessibility Considerations

### WCAG Compliance

**Color contrast:**
- Minimum 4.5:1 ratio for normal text
- Minimum 3:1 ratio for large text (18pt+)
- Test with contrast checking tools

**Alternative text:**
- Provide alt text for all images
- Describe charts and graphs in speaker notes
- Don't rely solely on visual information

**Readable fonts:**
- Sans-serif fonts for clarity
- Adequate size (24pt+ minimum)
- Avoid italics for long passages
- Sufficient line spacing

**Cognitive accessibility:**
- Clear, simple language
- Consistent layout and navigation
- Avoid flashing or rapid animations
- Provide handouts or digital copies

## Handouts and Leave-Behinds

### When to Provide Handouts

**Before presentation:**
- Detailed technical content requiring note-taking
- Reference materials for follow-along
- Risk: audience reads ahead instead of listening

**After presentation:**
- Summary of key points
- Contact information and resources
- Detailed appendix with supporting data
- Preferred approach to maintain attention

### Handout Design

**One-pager summary:**
- Key takeaways (3-5 points)
- Visual summary or infographic
- Contact information and next steps
- QR code to additional resources

**Detailed document:**
- Slide deck with detailed speaker notes
- Appendix with supporting data
- References and further reading
- Formatted for printing or digital distribution
