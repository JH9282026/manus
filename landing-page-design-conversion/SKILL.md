# Landing Page Design & Conversion

Design high-converting landing pages using proven layout patterns, persuasion principles, and systematic optimization through A/B testing.

## Overview

A landing page has one job: convert visitors into leads or customers. Unlike general web pages, landing pages eliminate distractions and guide every visitor toward a single action. Average landing page conversion rates are 2–5%, but optimized pages consistently achieve 10–25%+. This skill covers layout frameworks, CTA optimization, form design, social proof strategies, and systematic A/B testing methodology.

## Landing Page Anatomy

| Section | Position | Purpose | Impact on Conversion |
|---------|----------|---------|---------------------|
| Hero headline | Above fold, top | Communicate value proposition in <5 seconds | Critical — 80% of visitors read only the headline |
| Sub-headline | Below headline | Expand on headline, add specificity | High |
| Hero image/video | Above fold | Visual proof or context | +25% with relevant imagery |
| Primary CTA | Above fold | Convert ready visitors immediately | Critical |
| Social proof bar | Just below fold | Build credibility before detail section | +15–25% |
| Benefits section | Mid-page | Explain value with benefit-oriented copy | High |
| Objection handling | Mid-page | Address concerns (price, effort, risk) | +10–20% |
| Testimonials/case studies | Lower page | Prove results with real stories | +10–15% |
| Final CTA | Bottom of page | Capture visitors who needed full page to decide | High |
| Trust elements | Footer area | Security, guarantees, privacy | +5–10% |

## Headline Writing Framework

### Headline Formulas

| Formula | Template | Example | Conversion Impact |
|---------|----------|---------|-------------------|
| Outcome-focused | "Get [Desired Outcome] Without [Pain Point]" | "Get 10× More Leads Without Cold Calling" | High |
| Specific number | "[Number] [Audience] Use [Product] to [Benefit]" | "5,000+ Marketers Use Acme to Double Email ROI" | High |
| Question | "Want to [Achieve Goal]?" | "Want to Cut Your Monthly Cloud Bill in Half?" | Medium–High |
| How-to | "How to [Achieve Outcome] in [Timeframe]" | "How to Build a Sales Pipeline in 14 Days" | Medium |
| Negative | "Stop [Pain Point] — Start [Positive Outcome]" | "Stop Losing Customers — Start Retaining 95%" | High |

### Headline Testing Priority
1. **Test value proposition angle** first (what benefit do you lead with?)
2. **Test specificity** (numbers vs. general claims)
3. **Test emotional tone** (aspirational vs. pain-avoidance)
4. **Test length** (short punch vs. detailed headline)

## CTA Optimization

### CTA Button Design Rules

| Element | Best Practice | Anti-Pattern |
|---------|--------------|-------------|
| Color | High contrast to page background | Same color family as page |
| Size | Minimum 44×44px, ideally larger | Small, hard-to-tap button |
| Text | Action verb + benefit ("Start Free Trial") | Generic ("Submit", "Click Here") |
| Placement | Above fold + repeated at bottom | Only at bottom of long page |
| White space | Generous padding around button | Crowded, competing elements |
| Visual cues | Arrow, directional imagery pointing to CTA | No directional guidance |

### CTA Text A/B Test Results (Industry Averages)

| CTA Text | Relative Performance |
|----------|---------------------|
| "Get Started Free" | Baseline |
| "Start Free Trial" | +5–10% |
| "Get My Free [Thing]" | +10–20% (personalization effect) |
| "Submit" | -30% (generic) |
| "Buy Now" | -10% (too aggressive for top-of-funnel) |
| "See Pricing" | +15% (for product pages) |

## Form Optimization

### Field Count vs. Conversion Rate

| Number of Fields | Relative Conversion | Best For |
|-----------------|---------------------|----------|
| 1 (email only) | Baseline (highest) | Newsletter, content download |
| 2 (email + name) | -5% | Personalized follow-up |
| 3 (email + name + company) | -15% | B2B lead gen |
| 4–5 fields | -25% | Qualified lead gen |
| 6–7 fields | -40% | High-value services |
| 8+ fields | -50%+ | Avoid unless absolutely necessary |

### Form Design Best Practices
- Single column layout (not side-by-side fields)
- Labels above fields (not placeholder-only)
- Real-time inline validation (not post-submit errors)
- Progress indicator for multi-step forms
- Auto-fill support for common fields
- Mobile-friendly input types (email, tel, number)

## Social Proof Hierarchy

| Proof Type | Credibility | Effort to Obtain | Placement |
|-----------|-------------|-------------------|-----------|
| Customer logos (enterprise) | Very High | Medium | Below hero |
| Specific metrics ("4.8/5 from 2,340 reviews") | Very High | Low | Near CTA |
| Video testimonials | High | High | Mid-page |
| Written testimonials with photo + name + title | High | Medium | Mid-to-lower page |
| Star ratings aggregate | Medium–High | Low | Near CTA |
| Press mentions / "As seen in" | Medium–High | Medium | Below hero |
| User count ("Join 50,000+ teams") | Medium | Low | Headline or sub-headline |
| Generic quotes (no name/photo) | Low | Low | Avoid |

## A/B Testing Methodology

### Testing Priority (Impact × Ease)

| Element | Expected Impact | Test Difficulty | Priority |
|---------|----------------|-----------------|----------|
| Headline | 10–30% | Easy | 1 — Test first |
| CTA text and color | 5–20% | Easy | 2 |
| Social proof type | 10–25% | Medium | 3 |
| Hero image/video | 5–15% | Medium | 4 |
| Form length | 15–40% | Easy | 5 |
| Page layout | 10–20% | Hard | 6 |
| Pricing presentation | 5–30% | Medium | 7 |

### Statistical Significance Rules
- Minimum 200 conversions per variant (100 conversions per arm minimum)
- Run test for at least 7 days (full business cycle)
- Use 95% confidence level as standard
- Don't peek — decide sample size upfront, wait until complete
- Test one variable at a time for clear attribution

## Performance Requirements

| Metric | Target | Impact of Missing Target |
|--------|--------|--------------------------|
| Page load time | <2 seconds | -7% conversion per second of delay |
| Time to interactive | <3 seconds | Users can't interact with forms |
| Mobile responsiveness | 100% | 60%+ traffic is mobile |
| Core Web Vitals (all green) | LCP <2.5s, CLS <0.1, INP <200ms | Google Ads Quality Score and organic rankings |

## Best Practices

- **One page, one goal** — every landing page should have exactly one CTA (repeated, not multiple CTAs)
- **Remove navigation** — landing pages without site navigation convert 100% more
- **Match ad copy to headline** — visitors who see messaging continuity convert 2× more
- **Show the CTA above the fold** — don't make visitors scroll to find the action
- **Use real faces** — stock photo people reduce trust; real team/customer photos increase it
- **Test continuously** — even a "great" landing page can improve; run a new test every 2 weeks
