# Infographic Production Workflow

Manage the complete infographic production process from concept through delivery with structured phases, clear deliverables, and quality gates.

## Overview

Infographic production requires coordinating research, content strategy, visual design, and stakeholder review into a predictable workflow. This skill provides a 7-phase production framework with role assignments, timeline estimates, quality checkpoints, and handoff protocols. It ensures consistent output quality whether producing a single data visualization or a campaign of 20+ assets.

| Phase | Duration | Deliverable | Quality Gate | Reference |
|-------|----------|-------------|-------------|-----------|
| 1. Discovery & Brief | 2–3 days | Creative brief, raw data | Brief sign-off by stakeholder | `/references/discovery-design-phases.md` |

## Timeline Estimation Guide

| Complexity | Total Duration | Characteristics | Team Size |
|------------|---------------|-----------------|-----------|
| Simple | 1–2 weeks | Single topic, <10 data points, static | 2–3 people |
| Standard | 2–4 weeks | Multi-section, 10–30 data points, charts | 3–4 people |
| Complex | 4–6 weeks | Interactive, 30+ data points, animations | 4–6 people |
| Campaign | 6–10 weeks | Multiple infographics, consistent style | 5–8 people |

### Rush Timeline Adjustments
- **Rush (50% timeline):** Requires dedicated team, skip wireframe iteration, pre-approved brand assets
- **Expedited (75% timeline):** Parallel discovery/content, single review round
- **Standard:** Full process with 2 review rounds

## Phase 1: Discovery & Brief Framework

### Creative Brief Template
1. **Objective:** What should the viewer learn, feel, or do?
2. **Audience:** Who is the primary viewer? Technical level?
3. **Key message:** One sentence the viewer should remember
4. **Data sources:** Where is the data coming from? How reliable?
5. **Brand guidelines:** Colors, fonts, logo usage, tone
6. **Distribution:** Where will this appear? (social, print, web, presentation)
7. **Format:** Static, animated, interactive?
8. **Dimensions:** Platform-specific sizes needed
9. **Timeline:** Deadline and any milestone dates
10. **Success metrics:** How will effectiveness be measured?

### Data Collection Checklist
- ☐ Primary data sources identified and verified
- ☐ Data accuracy confirmed (cross-reference 2+ sources)
- ☐ Data is current (within 1–2 years unless historical)
- ☐ Permission to use data confirmed
- ☐ Units and scales standardized
- ☐ Outliers and anomalies documented

## Phase 2–3: Content Strategy & Wireframing

### Narrative Structure Options

| Structure | When to Use | Example |
|-----------|-------------|---------|
| Top-down hierarchy | Key finding → supporting data | Annual report summary |
| Timeline/chronological | Showing progression | History of technology |
| Compare/contrast | Two options or before/after | Product comparison |
| Problem → solution | Persuasive or marketing | Industry challenges report |
| Process/flow | Step-by-step sequence | How-to or workflow guide |
| Geographic/spatial | Location-based data | Market expansion map |

### Wireframe Review Criteria
- Visual flow follows reading pattern (Z or F pattern for Western audiences)
- Data hierarchy matches narrative importance
- White space is intentional, not leftover
- Call-to-action placement is prominent
- Responsive considerations for multi-format delivery

## Phase 4: Visual Design Quality Standards

| Element | Standard | Common Mistake |
|---------|----------|----------------|
| Chart type | Matches data relationship | Pie chart for >6 categories |
| Color palette | Brand-aligned, 3–5 colors max | Too many colors, poor contrast |
| Typography | 2 fonts max (heading + body) | Decorative fonts in data labels |
| Data labels | Clear, appropriately sized | Labels overlapping or truncated |
| Visual hierarchy | 3 levels (headline, subhead, body) | Everything same visual weight |
| Accessibility | 4.5:1 contrast ratio, colorblind-safe | Relying solely on color for meaning |
| Resolution | 300dpi print / 72dpi web / 2× retina | Low-res exports for print |

## Phase 5–6: Review & Finalization

### Review Feedback Protocol
1. **Round 1 (internal):** Design team + content lead review for accuracy and brand
2. **Round 2 (stakeholder):** Client/stakeholder reviews for messaging and data accuracy
3. **Final check:** Proofreading, data verification, file format validation
4. Maximum 2 major revision rounds included in standard timeline

### Final Deliverables Checklist
- ☐ Source file (AI/Figma/Sketch)
- ☐ Print PDF (300dpi, CMYK if applicable)
- ☐ Web-optimized PNG/JPEG (72dpi, sRGB)
- ☐ Social media crops (platform-specific dimensions)
- ☐ Animated version (MP4/GIF if applicable)
- ☐ Alt text and accessibility description
- ☐ Data sources document
- ☐ Brand assets used (fonts, colors, icons)

## Best Practices

- **Get data sign-off before design starts** — redesigning around changed data is the #1 timeline killer
- **Create a style tile** before full design — agree on visual direction with a small sample
- **Limit revision rounds** — define scope in creative brief to prevent scope creep
- **Version control everything** — use clear naming: `ProjectName_v2.1_YYYYMMDD`
- **Export all formats at once** — don't wait for requests; deliver the complete package
- **Document data sources** — always include a sources list for credibility and future updates
