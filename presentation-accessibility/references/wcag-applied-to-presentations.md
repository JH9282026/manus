## WCAG Applied to Presentations

The Web Content Accessibility Guidelines (WCAG) provide the foundation for digital accessibility, including presentations. While developed for web content, WCAG principles apply directly to presentation design.



### The Four WCAG Principles (POUR)

**Perceivable**: Information must be presentable in ways all users can perceive. This includes:
- Text alternatives for images
- Captions for multimedia
- Sufficient color contrast
- Content that doesn't rely solely on sensory characteristics

**Operable**: Interface components must be operable by all users:
- Keyboard accessibility
- Sufficient time to read content
- No content that causes seizures
- Clear navigation methods

**Understandable**: Information and interface operation must be understandable:
- Readable text
- Predictable functionality
- Input assistance for forms

**Robust**: Content must be robust enough for interpretation by assistive technologies:
- Compatible with screen readers
- Proper semantic structure
- Valid document structure



### WCAG Conformance Levels

**Level A (Minimum)**: Basic accessibility requirements that must be met for any accessible presentation.

**Level AA (Recommended)**: Addresses major barriers and is the standard for most legal requirements and organizational policies.

**Level AAA (Enhanced)**: Highest level of accessibility, providing the most inclusive experience but not always achievable for all content types.
