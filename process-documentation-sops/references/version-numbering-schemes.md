### Version Numbering Schemes

**Semantic Versioning:**
- **Major.Minor.Patch** (e.g., 2.3.1)
- Major: Significant changes affecting process execution
- Minor: Additions or modifications without major impact
- Patch: Corrections, typos, formatting

**Simple Versioning:**
- **Integer versions** (1, 2, 3) for major releases
- **Decimal versions** (1.1, 1.2) for updates

**Date-Based Versioning:**
- Uses effective date as version: v2026-03-01
- Useful for compliance-focused documents
