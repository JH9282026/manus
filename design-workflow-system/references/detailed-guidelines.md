## How to Use
### Starting a New Project

1. Begin with **Research & Inspiration** skills
2. Establish **UX Foundation** before visual design
3. Apply **Visual Design** skills systematically
4. Adapt with **Responsive & Platform Design**
5. Iterate using **Review & Iteration** until 9/10+
6. Prepare final **Delivery** assets

### Iteration Loop

```
┌─────────────────────────────────────────────────────────┐
│                   DESIGN ITERATION LOOP                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Design → Review → Score → If <9/10 → Iterate → Repeat │
│                              If ≥9/10 → Deliver         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Quality Gates

| Phase | Minimum Score | Action |
|-------|---------------|--------|
| Initial Draft | Any | Continue |
| After 1st Iteration | 6/10 | Address major issues |
| After 2nd Iteration | 7/10 | Refine details |
| After 3rd Iteration | 8/10 | Polish and perfect |
| Final Approval | 9/10+ | Ready for delivery |

## Best Practices
1. **Always gather inspiration first** - Use Dribbble, user examples, web research
2. **Document decisions** - Record why choices were made
3. **Test against rubric** - Score early and often
4. **Embrace iteration** - Quality emerges through refinement
5. **Consider accessibility** - Build inclusive designs from the start
