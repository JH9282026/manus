## Form Design and Validation Messages

Forms are critical interaction points. The copy supporting forms can dramatically impact completion rates.



### Form Field Labels

Labels should clearly identify what information is needed. Best practices include:

- Use natural language ("Email address" not "Email_Address")
- Indicate required vs. optional fields
- Add help text for ambiguous fields
- Consider placeholder text for format hints



### Inline Validation

Inline validation provides immediate feedback as users complete forms.

**Validation Message Examples:**

**Success:**
> ✓ "Email address looks good"
> ✓ "Strong password"

**Warning:**
> "This username may already be taken. Try adding numbers."

**Error:**
> "Enter a valid phone number (10 digits)"
> "This field is required"



### Form Error Summary

For longer forms, an error summary at the top helps users understand what needs fixing.

**Example:**
> "Please fix 2 errors:
> • Email address is invalid
> • Password is too short (minimum 8 characters)"
