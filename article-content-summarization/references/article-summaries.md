# Article and News Summaries

Create quick-read summaries for news and blog content.

---

## Bullet Point Format

### Structure

**Headline Summary** (1 sentence)
- What happened / main point

**Key Facts** (3-5 bullets)
- Who, what, when, where
- Key numbers or data
- Important quotes

**Why It Matters** (1-2 sentences)
- Context and implications

---

## News Article Summary

### Essential Elements

1. **Lead**: Core news in one sentence
2. **Details**: Supporting facts
3. **Quotes**: Key statements
4. **Context**: Background
5. **Impact**: Why this matters

### Example Format

```
HEADLINE: [Main point in <10 words]

• [Key fact 1]
• [Key fact 2]  
• [Key fact 3]

QUOTE: "[Notable quote]" - [Source]

CONTEXT: [Why this matters]
```

---

## Blog Post Summary

### Structure

**Main Thesis**
- Author's core argument

**Key Points**
- Supporting arguments
- Examples given
- Data cited

**Takeaways**
- Actionable insights
- Recommendations

---

## Newsletter Digest Format

### Per-Article

- **Headline** (linked)
- **One-sentence summary**
- **Why read**: Brief hook

### Multi-Article

- Category headers
- 2-3 sentence per article
- Links for full read

---

## Social Media Summaries

### Twitter/X Length

- 280 characters max
- Core point only
- Link to full article

### LinkedIn Length

- 150-200 words
- Professional angle
- Key takeaways
- Engagement hook

---

## Speed-Reading Format

### TLDR Style

```
TLDR: [One-sentence summary]

Key points:
• Point 1
• Point 2
• Point 3
```

### Time Estimate

Include original reading time and summary time:
- "Original: 8 min read | Summary: 1 min"
