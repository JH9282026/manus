# Expansion & Monetization

Strategies for revenue expansion and pricing optimization.

---

## Expansion Revenue

### Expansion Types
| Type | Description | Tactics |
|------|-------------|----------|
| Upselling | Higher tier/plan | Feature gates, value demonstration |
| Cross-selling | Additional products | Bundling, recommendations |
| Usage expansion | More consumption | Adoption programs, use case expansion |
| Seat expansion | More users | Team features, admin tools |

### Expansion Triggers
| Trigger | Response |
|---------|----------|
| Usage approaching limit | Upgrade prompt, tier recommendation |
| New use case identified | Cross-sell relevant product |
| Team growth | Seat expansion offer |
| Feature request | Premium feature introduction |
| Success milestone | Expansion conversation |

### Net Revenue Retention (NRR)
```
NRR = (Start MRR + Expansion - Contraction - Churn) / Start MRR
```

**Benchmarks:**
| NRR | Assessment |
|-----|------------|
| < 90% | Significant churn problem |
| 90-100% | Healthy but room to grow |
| 100-110% | Good expansion motion |
| 110-130% | Strong expansion |
| > 130% | Best-in-class |

## Pricing Strategy

### Pricing Models
| Model | Description | Best For |
|-------|-------------|----------|
| Per Seat | Per user pricing | Collaboration tools |
| Usage-Based | Pay for consumption | APIs, infrastructure |
| Flat Rate | Single price | Simple products |
| Tiered | Feature-based tiers | SaaS products |
| Hybrid | Combination models | Complex products |

### Value-Based Pricing Process
1. **Understand** customer value perception
2. **Quantify** value delivered
3. **Segment** by willingness to pay
4. **Price** to capture fair share of value
5. **Test** and iterate

### Pricing Page Optimization
| Element | Best Practice |
|---------|---------------|
| Tiers | 3-4 options (anchoring) |
| Recommended | Highlight preferred tier |
| Features | Clear differentiation |
| CTA | Strong action language |
| Annual vs. Monthly | Show annual savings |

## Packaging Strategy

### Package Design Principles
- **Simplicity**: Easy to understand
- **Differentiation**: Clear tier distinction
- **Value ladder**: Logical progression
- **Target matching**: Tiers match segments

### Feature Allocation
| Tier | Feature Approach |
|------|------------------|
| Free | Core value, limited usage |
| Basic | Essential features |
| Professional | Full individual features |
| Enterprise | Team, admin, support |

### Good-Better-Best Structure
| Tier | Target | Key Features |
|------|--------|---------------|
| Good | Entry level | Core functionality |
| Better | Growth | Popular features |
| Best | Power users | Advanced features |

## Monetization Optimization

### Revenue Levers
| Lever | Approach |
|-------|----------|
| Conversion rate | Improve free-to-paid |
| ARPU | Pricing, packaging, expansion |
| Retention | Reduce churn |
| Expansion | Upsell, cross-sell |
| Volume | More customers |

### Price Testing
- Test different price points
- A/B test pricing pages
- Grandfathering existing customers
- Geographic pricing
- Promotional offers

### Revenue Forecasting
```
Revenue = (Customers × ARPU) + Expansion - Churn
```

Cohort-based forecasting:
- New customer revenue
- Expansion by cohort
- Churn by cohort
- Net revenue by cohort over time
