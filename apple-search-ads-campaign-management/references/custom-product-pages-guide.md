# Custom Product Pages Guide for Apple Search Ads

Comprehensive guide to creating, optimizing, and using custom product pages (CPPs) for Apple Search Ads campaigns to increase conversion rates and campaign performance.

---

## What Are Custom Product Pages?

Custom product pages are alternative versions of your App Store product page that allow you to showcase different features, content, or messaging to specific audiences. Each custom product page can have unique:

- **Screenshots** (up to 10)
- **App Preview Videos** (up to 3)
- **Promotional Text** (up to 170 characters)
- **Deep Links** (iOS 18+, direct to specific in-app content)

**Elements that remain consistent across all pages:**
- App name
- App icon
- Subtitle
- App rating and reviews
- In-app purchase information
- Privacy details
- Description

**Limits:**
- Up to 35 custom product pages per app (plus default page = 36 total)
- Up to 70 additional versions across iPhone and iPad

## Creating Custom Product Pages

### Setup in App Store Connect

**Step 1: Access Custom Product Pages**
1. Log into App Store Connect
2. Select your app
3. Navigate to "App Store" tab
4. Click "Custom Product Pages" in left sidebar
5. Click "+" to create new custom product page

**Step 2: Configure Basic Settings**
1. **Name:** Internal name for your reference (not visible to users)
   - Use descriptive names: "Holiday 2026", "Calorie Tracking Focus", "Beginner Audience"
2. **Default Language:** Select primary language
3. **Availability:** Choose when page goes live (immediately or scheduled)

**Step 3: Add Localizations**
1. Click "Add Localization"
2. Select language/region
3. Repeat for all target markets

**Step 4: Create Assets for Each Localization**

For each language/region:

**Promotional Text (Optional but Recommended):**
- Up to 170 characters
- Appears above description
- Highlight key message, seasonal offer, or new feature
- Can be updated without app review

**Screenshots:**
- **Required:** At least 3 screenshots
- **Maximum:** 10 screenshots
- **Sizes:** Must match device requirements
  - iPhone: 6.7" display (1290 x 2796 pixels) or 6.5" display (1242 x 2688 pixels)
  - iPad: 12.9" display (2048 x 2732 pixels)
- **Format:** PNG or JPG
- **Order matters:** First 3 screenshots are most visible

**App Preview Videos (Optional):**
- **Maximum:** 3 videos
- **Duration:** 15-30 seconds recommended (max 30 seconds)
- **Sizes:** Must match device requirements
  - iPhone: 1920 x 1080 pixels (landscape) or 1080 x 1920 pixels (portrait)
  - iPad: 1920 x 1080 pixels (landscape) or 1080 x 1920 pixels (portrait)
- **Format:** .mov, .m4v, or .mp4
- **Poster Frame:** Select compelling frame for thumbnail
- **Order matters:** First video auto-plays

**Step 5: Add Deep Link (iOS 18+ Only, Optional)**
1. Enter deep link URL
2. Deep link directs users to specific in-app content after install
3. Format: `yourapp://specific-content`
4. Test thoroughly before submission

**Step 6: Submit for Review**
1. Review all assets
2. Click "Submit for Review"
3. Wait for Apple approval (typically 24-48 hours)
4. Monitor status in App Store Connect

**Step 7: Publish**
1. Once approved, page is available for use
2. Can be used immediately in Apple Search Ads
3. Can also be promoted via URL outside of ads

## Custom Product Page Best Practices

### Screenshot Optimization

**1. First Three Screenshots Are Critical**
- Most users only see first 3 before deciding to install
- Put most compelling content first
- Lead with strongest value proposition

**2. Use Text Overlays**
- Add descriptive text to screenshots
- Highlight key features and benefits
- Keep text concise and readable
- Use high contrast for readability

**3. Show, Don't Tell**
- Use actual app interface (not just marketing graphics)
- Show features in action
- Include realistic use cases

**4. Maintain Visual Consistency**
- Use consistent color scheme
- Use consistent fonts and styling
- Create cohesive visual story

**5. Optimize for Vertical Viewing**
- Most users view in portrait mode
- Design screenshots for vertical orientation
- Ensure key elements are visible in portrait

**6. Include Social Proof**
- Show ratings/reviews in screenshots (if strong)
- Include user testimonials
- Display awards or recognition

**7. Highlight Unique Selling Points**
- What makes your app different?
- Why choose your app over competitors?
- What problem does it solve?

### App Preview Video Optimization

**1. Hook in First 3 Seconds**
- Capture attention immediately
- Show most compelling feature first
- Use motion and visual interest

**2. Keep It Short**
- 15-30 seconds is ideal
- Users have short attention spans
- Focus on 1-2 key features

**3. Show Real App Usage**
- Demonstrate actual app interface
- Show realistic user interactions
- Avoid overly produced marketing videos

**4. No Audio Required**
- Many users watch without sound
- Use text overlays to convey message
- Ensure video makes sense without audio

**5. Optimize Poster Frame**
- Choose compelling thumbnail
- Should entice users to play video
- Avoid generic or boring frames

**6. Test Different Approaches**
- Feature demo vs. user testimonial
- Fast-paced vs. slower, detailed
- Different features highlighted

### Promotional Text Optimization

**1. Lead with Value Proposition**
- What's the main benefit?
- Why should user install now?
- Be specific and compelling

**2. Create Urgency (If Appropriate)**
- "Limited time offer"
- "New feature just launched"
- "Join 10M+ users"

**3. Highlight What's New**
- "Now with [new feature]"
- "Just added: [feature]"
- "Introducing [feature]"

**4. Use Numbers and Specifics**
- "Track 100+ exercises"
- "Learn 5,000+ words"
- "Edit photos in seconds"

**5. Keep It Concise**
- 170 characters max
- Get to the point quickly
- Every word should add value

**6. Test Different Messages**
- Feature-focused vs. benefit-focused
- Emotional vs. rational
- Different value propositions

### Deep Link Optimization (iOS 18+)

**1. Direct to Relevant Content**
- Match deep link to campaign theme
- Send users to specific feature they searched for
- Reduce friction to value

**2. Test Thoroughly**
- Verify deep link works on fresh install
- Test on different iOS versions
- Test on different devices

**3. Have Fallback**
- If deep link fails, ensure good default experience
- Don't break user experience with broken links

**4. Use for Onboarding**
- Skip generic onboarding for targeted users
- Show relevant tutorial for specific feature
- Personalize first-run experience

## Using Custom Product Pages in Apple Search Ads

### Campaign Setup

**Search Results Ads:**
1. Create or edit ad group
2. In "Ad Variation" section, select "Custom Product Page"
3. Choose approved custom product page from dropdown
4. Save ad group
5. Ads will now direct to custom product page instead of default

**Today Tab Ads:**
1. Create Today Tab campaign
2. Create ad
3. **Required:** Select approved custom product page
4. Custom product page assets will animate in ad background
5. Submit ad for Apple approval
6. Wait for approval before activating campaign

**Search Tab Ads:**
1. Create or edit ad group
2. Select custom product page (optional)
3. Save ad group

**Product Page Ads:**
1. Create or edit ad group
2. Select custom product page (optional)
3. Save ad group

### Matching Custom Product Pages to Campaigns

**Best Practices:**

1. **Brand Campaigns → Default Product Page**
   - Users already know your app
   - Show comprehensive overview
   - Default page is usually best

2. **Category Campaigns → Theme-Specific Custom Product Pages**
   - Match CPP to keyword theme
   - "Calorie counting" keywords → Calorie tracking CPP
   - "Workout plans" keywords → Workout CPP

3. **Competitor Campaigns → Competitive Positioning Custom Product Pages**
   - Highlight advantages over competitor
   - Address why users should switch
   - Show unique features

4. **Discovery Campaigns → Default Product Page or Broad Custom Product Page**
   - Unknown what users are looking for
   - Show broad value proposition
   - Highlight most popular features

5. **Today Tab Campaigns → Seasonal/Event Custom Product Pages**
   - Leverage visual impact of Today Tab
   - Align with current events or seasons
   - Create compelling animated background

## Measuring Custom Product Page Performance

### App Analytics (App Store Connect)

**Access:**
1. Log into App Store Connect
2. Select your app
3. Navigate to "App Analytics"
4. Select "Product Page" tab
5. Filter by custom product page

**Key Metrics:**

| Metric | What It Measures | Target |
|--------|------------------|--------|
| **Impressions** | How many users saw the page | Higher is better (reach) |
| **Product Page Views** | How many users viewed full page | Higher is better (interest) |
| **Downloads** | How many users installed | Higher is better (conversions) |
| **Conversion Rate** | Downloads ÷ Product Page Views | **Key metric** - aim for 2.5%+ improvement vs default |
| **Proceeds** | Revenue from users who came via this page | Higher is better (monetization) |

**Analysis:**
- Compare custom product page conversion rate to default page
- Identify which custom product pages perform best
- Analyze by source (Apple Search Ads, organic, etc.)
- Track trends over time

### Apple Search Ads Dashboard

**Campaign-Level Analysis:**
1. Navigate to campaign using custom product page
2. Review standard metrics:
   - Impressions
   - Taps
   - Tap-Through Rate
   - Installs
   - Conversion Rate (Installs ÷ Taps)
   - Cost Per Tap
   - Cost Per Acquisition

**Comparison:**
- Compare campaigns using custom product pages vs default page
- Look for conversion rate improvements
- Assess if higher conversion rate justifies any increase in CPT
- Calculate overall CPA impact

**A/B Testing:**
- Run parallel campaigns with different custom product pages
- Keep all other variables constant (keywords, bids, targeting)
- Compare conversion rates
- Scale winner, pause loser

## Custom Product Page A/B Testing

### What to Test

**1. Screenshot Order**
- Test different feature priorities
- Test different visual hierarchies
- Identify which features drive highest conversions

**2. Screenshot Content**
- Test different text overlays
- Test different visual styles
- Test feature demos vs. benefit statements

**3. App Preview Videos**
- Test video vs. no video
- Test different video content
- Test different video lengths

**4. Promotional Text**
- Test different value propositions
- Test feature-focused vs. benefit-focused
- Test urgency messaging vs. evergreen

**5. Feature Emphasis**
- Test highlighting different features
- Test different use cases
- Test different audience messaging

### Testing Methodology

**Setup:**
1. Create 2+ custom product pages with single variable changed
2. Create separate campaigns or ad groups for each variant
3. Ensure all other variables are identical:
   - Same keywords
   - Same bids
   - Same targeting
   - Same budget allocation

**Run Test:**
1. Launch campaigns simultaneously
2. Run for minimum 2 weeks (or until statistical significance)
3. Ensure each variant gets sufficient traffic (minimum 100 installs per variant)

**Analyze:**
1. Compare conversion rates (primary metric)
2. Compare CPA (secondary metric)
3. Calculate statistical significance
4. Declare winner

**Scale:**
1. Pause losing variant
2. Allocate full budget to winning variant
3. Apply learnings to other custom product pages
4. Start next test

### Statistical Significance

Don't declare a winner too early. Ensure statistical significance:

**Minimum Sample Size:**
- At least 100 installs per variant
- At least 2 weeks of data
- Consistent performance (not just a lucky day)

**Significance Calculators:**
- Use online A/B test calculators
- Input: Visitors (taps), Conversions (installs) for each variant
- Look for 95%+ confidence level

**Example:**
- Variant A: 1,000 taps, 250 installs (25% conversion rate)
- Variant B: 1,000 taps, 300 installs (30% conversion rate)
- Difference: 5 percentage points (20% relative improvement)
- If statistically significant at 95%+ confidence → Variant B wins

## Custom Product Page Compliance

### Apple Advertising Policies

**Prohibited Content:**
- Violent, offensive, or sexually explicit content
- Misleading claims or false information
- Pricing or ranking claims (in Today Tab ads)
- Content that violates App Store Review Guidelines

**Required:**
- Accurate representation of app functionality
- Screenshots from actual app (not mockups)
- Compliance with all App Store guidelines

**Today Tab Specific:**
- No pricing, offers, or ranking claims in screenshots or promotional text
- No violent, offensive, or inappropriate content
- Must be localized in default language of target region
- Must include iPhone assets (if targeting iPhone)

### Review Process

**Timeline:**
- Typically 24-48 hours for approval
- Can be longer during peak times
- Plan ahead for time-sensitive campaigns

**Rejection Reasons:**
- Prohibited content
- Misleading screenshots
- Poor quality assets
- Non-compliance with guidelines

**If Rejected:**
1. Review rejection reason in App Store Connect
2. Revise assets to address issue
3. Resubmit for review
4. Contact Apple support if unclear

## Custom Product Page Workflow

### Planning Phase
1. Define campaign strategy
2. Identify which campaigns need custom product pages
3. Determine custom product page themes (keyword-based, audience-based, seasonal, etc.)
4. Create custom product page plan (how many, for what purposes)

### Creation Phase
1. Design screenshots for each custom product page
2. Create app preview videos (if using)
3. Write promotional text
4. Set up deep links (if using)
5. Create custom product pages in App Store Connect
6. Submit for review

### Launch Phase
1. Wait for approval
2. Create/update Apple Search Ads campaigns
3. Assign custom product pages to appropriate ad groups
4. Launch campaigns

### Optimization Phase
1. Monitor performance in App Analytics and Apple Ads dashboard
2. Compare custom product page performance to default page
3. A/B test variations
4. Iterate and improve
5. Scale winners, pause losers

### Maintenance Phase
1. Update custom product pages seasonally
2. Refresh for new feature launches
3. Keep promotional text current
4. Archive outdated custom product pages
5. Continuously test and optimize
