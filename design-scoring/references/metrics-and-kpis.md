# Design Metrics and KPIs

## Overview

Design metrics quantify design effectiveness and impact. Key Performance Indicators (KPIs) track progress toward business and user goals, providing objective evidence of design value.

---

## User Experience Metrics

**Task Success Rate**: Percentage of users completing intended tasks. Measures core usability.

**Time on Task**: How long users take to complete tasks. Shorter is generally better, but context matters.

**Error Rate**: Frequency of user errors. Includes mistakes, wrong paths, and failed attempts.

**Learnability**: How quickly new users become proficient. Measure time to first success or success rate over time.

**Memorability**: How easily returning users remember how to use the interface. Measure success rate after time away.

**Satisfaction**: User-reported satisfaction via surveys (SUS, CSAT, NPS). Subjective but valuable.

---

## Business Impact Metrics

**Conversion Rate**: Percentage of users completing desired action (purchase, signup, etc.). Direct business impact.

**Revenue per User**: Average revenue generated per user. Shows economic value of design improvements.

**Customer Lifetime Value (CLV)**: Total revenue from customer over relationship. Good design increases CLV.

**Churn Rate**: Percentage of users who stop using product. Good UX reduces churn.

**Support Ticket Volume**: Number of support requests. Better design reduces support needs.

**Net Promoter Score (NPS)**: Likelihood users will recommend product. Measures overall satisfaction and loyalty.

---

## Engagement Metrics

**Active Users**: Daily Active Users (DAU), Weekly Active Users (WAU), Monthly Active Users (MAU). Shows product stickiness.

**Session Duration**: Average time users spend in product. Longer may indicate engagement or confusion—context matters.

**Session Frequency**: How often users return. Higher frequency indicates habit formation.

**Feature Adoption**: Percentage of users using specific features. Measures discoverability and value.

**Depth of Use**: How many features users engage with. Indicates product understanding and value realization.

**Return Rate**: Percentage of users who return after first visit. Measures initial experience quality.

---

## Performance Metrics

**Page Load Time**: Time until page is interactive. Target: <3 seconds. Affects bounce rate and conversions.

**Time to Interactive (TTI)**: When page becomes fully interactive. Critical for user experience.

**First Contentful Paint (FCP)**: When first content appears. Affects perceived performance.

**Largest Contentful Paint (LCP)**: When main content loads. Target: <2.5 seconds.

**Cumulative Layout Shift (CLS)**: Visual stability during loading. Target: <0.1.

**Core Web Vitals**: Google's performance metrics (LCP, FID, CLS). Affects SEO and user experience.

---

## Accessibility Metrics

**WCAG Conformance Level**: A, AA, or AAA. Most organizations target AA.

**Automated Test Pass Rate**: Percentage of automated accessibility tests passed. Baseline metric.

**Keyboard Navigation Success**: Percentage of tasks completable via keyboard only.

**Screen Reader Compatibility**: Success rate for screen reader users. Requires user testing.

**Color Contrast Compliance**: Percentage of text meeting contrast requirements.

**Assistive Technology Usage**: Percentage of users using assistive technologies. Informs prioritization.

---

## Design System Metrics

**Component Adoption Rate**: Percentage of product using design system components. Measures system success.

**Design Consistency Score**: Percentage of designs following system guidelines. Audit-based metric.

**Time to Design**: Average time to design new features. Should decrease as system matures.

**Time to Develop**: Average development time for designs. Good systems reduce dev time.

**Design Debt**: Number of non-system components or inconsistencies. Track and reduce over time.

**System Contribution Rate**: Number of new components or improvements contributed. Measures system health.
