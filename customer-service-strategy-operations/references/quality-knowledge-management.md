# Quality & Knowledge Management

## Quality Assurance Program Design

### QA Program Structure

| Component | Description | Frequency |
|-----------|-------------|-----------|
| Random evaluations | Score sample of interactions | Weekly per agent |
| Targeted evaluations | Focus on specific issues | As needed |
| Calibration sessions | Align scorer interpretation | Monthly |
| Agent self-assessments | Self-evaluation against rubric | Monthly |
| Peer reviews | Agent-to-agent feedback | Quarterly |
| Customer correlation | Compare QA scores to CSAT | Monthly |

### Evaluation Volume Guidelines

| Team Size | Evaluations per Agent/Month | Total Monthly Evaluations |
|-----------|---------------------------|--------------------------|
| <20 agents | 8-10 | 160-200 |
| 20-50 agents | 5-8 | 100-400 |
| 50-100 agents | 4-6 | 200-600 |
| 100+ agents | 3-5 | 300-500+ |

### Calibration Process

1. Select sample interactions (include borderline cases)
2. All evaluators score independently
3. Compare scores and discuss discrepancies
4. Establish consensus on scoring standards
5. Update rubric guidance based on discussion
6. Track inter-rater reliability (target >90% agreement)

### Coaching from QA Data

| QA Finding | Coaching Approach |
|-----------|-------------------|
| Knowledge gaps | Targeted training, knowledge base review |
| Process non-compliance | Procedure review, system workflow adjustment |
| Communication issues | Role-playing, call listening, mentoring |
| Empathy deficit | Customer perspective exercises, story sharing |
| Efficiency issues | Time management training, tool proficiency |

## Knowledge-Centered Service (KCS)

### KCS Methodology

| Principle | Description |
|-----------|-------------|
| Capture | Create knowledge as a byproduct of solving problems |
| Structure | Use consistent templates for all knowledge articles |
| Reuse | Search and apply existing knowledge before creating new |
| Improve | Update articles based on feedback and usage data |

### KCS Article Lifecycle

1. **Draft** — Agent creates article while solving issue
2. **Review** — Subject matter expert validates accuracy
3. **Publish** — Article available to agents and/or customers
4. **Maintain** — Regular review cycle based on feedback flags
5. **Archive** — Retire outdated content

### Knowledge Base Structure

| Content Type | Audience | Format | Review Cycle |
|-------------|----------|--------|-------------|
| How-to articles | Customers + Agents | Step-by-step with screenshots | Quarterly |
| Troubleshooting guides | Agents | Decision tree format | Monthly |
| FAQ | Customers | Question/Answer | Quarterly |
| Policy documents | Agents | Reference format | Semi-annually |
| Release notes | All | Changelog format | Per release |

### Knowledge Health Metrics

| Metric | Target | Measurement |
|--------|--------|-------------|
| Article usage rate | >70% of contacts reference KB | System tracking |
| Article helpfulness | >80% rated helpful | User feedback |
| Content coverage | >90% of contact reasons | Gap analysis |
| Article accuracy | >95% confirmed accurate | QA sampling |
| Time to publish | <24 hours for new issues | Workflow tracking |
| Content freshness | <10% articles overdue for review | Audit schedule |
