# Validation Methods

Approaches for testing business model hypotheses and achieving product-market fit.

---

## Hypothesis-Driven Validation

### Types of Hypotheses

| Type | Question | Testing Approach |
|------|----------|------------------|
| Problem | Do customers have this problem? | Interviews, surveys |
| Solution | Does our solution address the problem? | Prototypes, MVPs |
| Customer | Are these the right customers? | Segment testing |
| Channel | Can we reach customers this way? | Marketing experiments |
| Revenue | Will customers pay this much? | Pricing tests |
| Growth | Can we scale acquisition? | Channel testing |

### Hypothesis Template

```
We believe that [customer segment]
Will [desired behavior]
Because [reason/value proposition]

We will test this by [experiment]
And measure [metric]

We are right if [success criteria]
```

## Customer Discovery

### Interview Best Practices

| Principle | Application |
|-----------|-------------|
| Talk about their life | Not your idea |
| Ask about specifics | Past behavior, not hypotheticals |
| Listen more than talk | 80/20 rule |
| Follow the emotion | Dig into strong reactions |
| Avoid leading questions | Don't suggest answers |

### Key Discovery Questions

**Problem Questions**:
- What's the hardest part about [context]?
- Tell me about the last time you [relevant activity]
- What solutions have you tried?
- What do you wish existed?

**Solution Questions**:
- Walk me through how you would use this
- What would make you switch from your current solution?
- What features are must-have vs nice-to-have?

**Willingness-to-Pay Questions**:
- How much are you spending on this problem today?
- What would be a reasonable price for a solution?
- What would make this a no-brainer purchase?

## Minimum Viable Product (MVP)

### MVP Types

| Type | Description | Speed | Fidelity |
|------|-------------|-------|----------|
| Concierge | Manual delivery of service | Fast | Low |
| Wizard of Oz | Human-powered "automation" | Fast | Medium |
| Landing Page | Marketing page before product | Very Fast | Low |
| Video Explainer | Demonstrate concept | Fast | Low |
| Prototype | Functional subset | Medium | Medium |
| Single Feature | One core capability | Medium | High |
| Crowdfunding | Pre-sell before building | Fast | Low |

### MVP Design Principles

1. **Identify the riskiest assumption** — What must be true for success?
2. **Design the smallest test** — Minimum to validate/invalidate
3. **Set success criteria upfront** — Know what "works" means
4. **Build just enough** — No more than needed for learning
5. **Measure what matters** — Actionable metrics only

## Product-Market Fit

### Indicators of Product-Market Fit

| Signal | Description |
|--------|-------------|
| Retention | Users keep coming back |
| Organic Growth | Word-of-mouth referrals |
| Usage Depth | Users adopt multiple features |
| Willingness to Pay | Customers convert to paid |
| Low Churn | Customers stay subscribed |
| Sean Ellis Test | >40% would be "very disappointed" if product disappeared |

### Sean Ellis Survey

**Key Question**: "How would you feel if you could no longer use [product]?"

| Response | Indication |
|----------|------------|
| Very disappointed | Strong fit |
| Somewhat disappointed | Partial fit |
| Not disappointed | Poor fit |

**Target**: >40% "very disappointed" indicates product-market fit

### Measuring Product-Market Fit

| Metric | Target Range | Indication |
|--------|--------------|------------|
| Week 1 Retention | >40% | Early engagement |
| Month 1 Retention | >20% | Core value delivery |
| NPS | >40 | Customer satisfaction |
| Organic Traffic | >50% | Word of mouth |
| Sean Ellis Score | >40% | Product-market fit |

## Experimentation Framework

### Experiment Design

| Element | Description |
|---------|-------------|
| Hypothesis | What you believe to be true |
| Experiment | How you'll test it |
| Metrics | What you'll measure |
| Success Criteria | What determines success |
| Timeline | Duration of test |

### Validation Prioritization

| Factor | Assessment |
|--------|------------|
| Risk | How uncertain is this assumption? |
| Impact | How critical to the model? |
| Speed | How quickly can we test? |
| Cost | How expensive to test? |

**Priority Formula**: (Risk × Impact) / (Speed × Cost)

### Learning Documentation

| Element | Content |
|---------|---------|
| Hypothesis | What we believed |
| Experiment | What we did |
| Results | What we observed |
| Learning | What we concluded |
| Action | What we're doing next |
