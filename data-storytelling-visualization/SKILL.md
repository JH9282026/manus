# Data Storytelling & Visualization

Combine data, visuals, and narrative to communicate insights in a compelling, understandable way that drives action.

## Overview

Transform raw numbers into meaningful stories by weaving together accurate data analysis, effective visualization, and coherent narrative structure. Apply frameworks for audience analysis, narrative design, chart selection, and visual communication that make data accessible and actionable.

## The Three Pillars

| Pillar | Role | Key Considerations |
|--------|------|-------------------|
| **Data** | Foundation of truth | Accuracy, relevance, proper analysis |
| **Visuals** | Translation layer | Chart selection, design, accessibility |
| **Narrative** | Meaning and context | Structure, audience, call to action |

## Audience-First Communication

| Audience | Focus | Detail Level | Format |
|----------|-------|-------------|--------|
| C-Suite/Board | Business impact, decisions | High-level, 3-5 key metrics | 1-page summary, dashboard |
| Managers | Operational insights, trends | Moderate, actionable detail | Slide deck, report |
| Analysts | Methodology, deep data | Technical, full detail | Detailed report, notebook |
| General public | Simplified story, relatability | Minimal, visual-heavy | Infographic, article |
| Investors | Growth, risk, opportunity | Financial metrics, projections | Pitch deck, prospectus |

## Chart Selection Guide

| Data Relationship | Recommended Chart | Avoid |
|------------------|-------------------|-------|
| Comparison across categories | Bar chart (horizontal for many) | Pie chart (>5 slices) |
| Trend over time | Line chart | Area chart (unless stacking) |
| Part-to-whole | Stacked bar, treemap | 3D pie chart |
| Distribution | Histogram, box plot | Stem-and-leaf |
| Correlation | Scatter plot | Line chart |
| Geographic | Choropleth map, bubble map | Mercator distortion |
| Ranking | Horizontal bar, lollipop | Vertical bar (many items) |
| Flow/Process | Sankey diagram, funnel | Circular flow |

## Narrative Structures for Data

### The Situation-Complication-Resolution (SCR)

1. **Situation**: Establish context ("Last quarter, we targeted 20% growth...")
2. **Complication**: Present the challenge/surprise ("However, churn increased 35%...")
3. **Resolution**: Deliver insight and recommendation ("Analysis reveals... We should...")

### The What-So What-Now What

1. **What**: Present the data finding ("Customer acquisition cost rose 40%")
2. **So What**: Explain why it matters ("At this rate, payback period exceeds 18 months")
3. **Now What**: Recommend action ("Shift budget to retention, targeting 15% reduction")

### The Data Narrative Arc

1. **Hook** — Start with a surprising or compelling data point
2. **Context** — Provide background and set expectations
3. **Rising tension** — Show the problem or opportunity emerging
4. **Climax** — Present the key insight or turning point
5. **Resolution** — Deliver the recommendation
6. **Call to action** — Specify what the audience should do next

## Visual Design Principles

| Principle | Application |
|-----------|-------------|
| Data-ink ratio | Maximize data, minimize decoration |
| Pre-attentive attributes | Use color, size, position for emphasis |
| Gestalt principles | Group related elements, separate unrelated |
| Progressive disclosure | Summary first, detail on demand |
| Accessibility | Colorblind-safe palettes, text alternatives |
| Consistency | Same colors, scales, fonts throughout |

## Common Mistakes to Avoid

1. Starting with the chart instead of the message
2. Showing all available data instead of curating
3. Using misleading scales (truncated axes, dual axes)
4. Choosing chart type based on aesthetics rather than data
5. Neglecting mobile/projected display readability
6. Forgetting to state the "so what"
7. Including too many insights in one visualization
