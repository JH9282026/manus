# Narrative Frameworks & Delivery

## Data Story Structures

### The McKinsey Pyramid (MECE)

**Structure**: Start with the answer, then support with evidence

1. **Governing thought** — State the main conclusion/recommendation upfront
2. **Key arguments** — 3-4 supporting points (mutually exclusive, collectively exhaustive)
3. **Supporting data** — Evidence backing each argument

**Best for**: Executive presentations, consulting-style recommendations

### The Hero's Journey for Data

Adapt Joseph Campbell's structure for data narratives:

1. **Ordinary World** — Current state, business as usual
2. **Call to Adventure** — Data reveals an opportunity or threat
3. **Crossing the Threshold** — Deep dive into the data
4. **Tests and Revelations** — Surprising findings, pattern discovery
5. **The Ordeal** — The biggest challenge or most critical insight
6. **The Reward** — The key recommendation
7. **The Return** — Implementation plan, expected outcomes

**Best for**: Change management, persuading skeptical audiences

### The Newspaper Model

**Structure**: Most important first, progressive detail

1. **Headline** — One-sentence key finding
2. **Lead paragraph** — Who, what, when, where, why summary
3. **Body** — Supporting data and analysis
4. **Background** — Context, methodology, caveats

**Best for**: Written reports, email summaries, dashboards

## Delivery Formats

### Executive Summary (1-Page)

| Section | Length | Content |
|---------|--------|---------|
| Title | 1 line | Clear, specific (not "Q3 Report") |
| Key finding | 2-3 sentences | Bottom-line insight with number |
| Supporting evidence | 3-4 bullets | Data points backing the finding |
| Visualization | 1 chart | Most impactful chart only |
| Recommendation | 2-3 sentences | Specific next steps with owners |
| Risk/caveat | 1 sentence | Key limitation or risk |

### Slide Deck Structure

| Slide | Purpose | Design |
|-------|---------|--------|
| 1 - Title | Set context | Title, subtitle, date |
| 2 - Executive summary | Key takeaways | 3-5 bullet points |
| 3 - Context | Background | Brief text + one chart |
| 4-8 - Findings | Data stories | One insight per slide |
| 9 - Recommendations | Actions | Numbered, specific, owned |
| 10 - Next steps | Timeline | Who does what by when |
| Appendix | Supporting detail | Methodology, full data tables |

**Slide Design Rules**:
- One message per slide
- Chart title = the insight (not "Revenue by Quarter" but "Revenue grew 23% driven by APAC")
- Maximum 3 data elements per visualization
- Annotation over decoration
- Speaker notes for verbal narrative

### Written Report Structure

1. **Executive Summary** (1 page) — Key findings and recommendations
2. **Introduction** (0.5 pages) — Purpose, scope, methodology
3. **Key Findings** (3-5 pages) — Organized by theme, not by data source
4. **Detailed Analysis** (5-10 pages) — Supporting evidence and methodology
5. **Recommendations** (1-2 pages) — Prioritized actions with rationale
6. **Appendix** — Methodology details, full data tables, glossary

## Persuasion Techniques for Data

### Anchoring
Present a reference point before your key number to frame perception:
- "Industry average is 3%. We achieved 7%."
- "Last year we lost $2M to churn. This initiative saves $1.5M."

### Social Proof
Use comparative data to validate recommendations:
- "Companies that adopted this approach saw 40% improvement"
- "3 of 5 market leaders have already implemented this strategy"

### Loss Framing
Frame inaction as losing rather than action as gaining:
- "Without intervention, we project $5M in lost revenue" (more powerful)
- vs. "With this initiative, we can gain $5M in revenue"

### Specificity
Specific numbers are more credible than round numbers:
- "Customer satisfaction improved 23.4%" beats "improved about 25%"
- Include confidence intervals when appropriate

### The Rule of Three
Group insights into three key points:
- Three findings
- Three recommendations
- Three supporting data points per finding
- Audiences remember three things; more gets lost

## Handling Uncertainty and Limitations

### Communicating Uncertainty
- Use confidence intervals instead of point estimates
- Show ranges: "Revenue will likely fall between $10M and $12M"
- Use probability language: "There is a 70% chance of exceeding target"
- Visualize uncertainty with error bars, fan charts, or probability distributions

### Disclosure Best Practices
- State sample size and methodology clearly
- Acknowledge data quality limitations
- Distinguish correlation from causation explicitly
- Note time period and whether findings are generalizable
- Present alternative interpretations when evidence is ambiguous
