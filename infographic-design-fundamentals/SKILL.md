# Infographic Design Fundamentals

Apply core design principles to create clear, compelling, and data-accurate infographics that communicate complex information visually.

## Overview

Effective infographics transform data into visual stories that are understood 60,000× faster than text alone. But poor infographic design — cluttered layouts, misleading charts, decorative excess — does more harm than good. This skill covers the fundamental design principles every infographic needs: visual hierarchy, chart selection, color theory, typography, layout composition, and data integrity. Design serves communication, not decoration.

## Visual Hierarchy Framework

| Level | Element | Size | Treatment | Purpose |
|-------|---------|------|-----------|---------|
| 1 | Main title | 36–48pt | Bold, accent color | Attract attention, state topic |
| 2 | Section headers | 24–32pt | Semi-bold, primary color | Organize major sections |
| 3 | Data callouts | 48–72pt (number) | Bold number + small label | Highlight key statistics |
| 4 | Subsection text | 14–18pt | Regular, secondary color | Explain sections |
| 5 | Body/annotation | 10–14pt | Light, neutral gray | Provide context and detail |
| 6 | Source/credits | 8–10pt | Italic, light gray | Attribution and credibility |

### Hierarchy Rules
- Every infographic needs exactly 3 levels of visual importance (primary, secondary, tertiary)
- The most important data point should be visible from 6 feet away
- Use size, color, weight, and whitespace — not all at once, but deliberately
- Guide the eye from top-left to bottom-right (for Western audiences) or center-outward

## Chart Selection Guide

| Data Relationship | Chart Type | Max Items | Common Mistake |
|------------------|------------|-----------|----------------|
| Part of whole | Donut or pie | ≤5 segments | Pie chart with 10+ segments |
| Comparison | Horizontal bar | ≤15 bars | Vertical bars with long labels |
| Trend over time | Line chart | ≤5 lines | Too many overlapping lines |
| Ranking | Sorted horizontal bar | ≤20 | Unsorted bars |
| Distribution | Histogram | — | Using bar chart instead |
| Correlation | Scatter plot | ≤3 series | Overplotting with 1,000+ points |
| Composition over time | Stacked area | ≤5 series | Stacked bar for continuous data |
| Single metric | Big number + icon | 1 | Unnecessary chart for one number |

### Data Integrity Rules
- **Start bar chart axes at zero** — truncated axes exaggerate differences
- **Use consistent scales** — comparing charts with different scales is misleading
- **Label all axes and units** — "Revenue" is ambiguous; "Revenue ($M, FY2026)" is clear
- **Don't use 3D charts** — 3D distorts perception; always use 2D
- **Cite data sources** — every data point needs attribution for credibility
- **Use area proportionally** — bubble/icon sizes must scale by area, not diameter

## Color Theory for Infographics

### Color Palette Rules

| Rule | Implementation | Why |
|------|---------------|-----|
| Maximum 5 colors | 1 primary, 1 accent, 3 supporting | More colors = visual noise |
| 60-30-10 rule | 60% dominant, 30% secondary, 10% accent | Creates balanced hierarchy |
| Use color meaningfully | Red = negative, Green = positive, Gray = neutral | Color carries inherent meaning |
| Colorblind-safe | Use patterns + color, avoid red-green only distinctions | 8% of males are colorblind |
| Consistent throughout | Same color = same category across all charts | Prevents confusion |
| Background contrast | 4.5:1 minimum for text readability | WCAG accessibility standard |

### Sequential vs. Diverging Palettes

| Palette Type | Use Case | Example |
|-------------|----------|---------|
| Sequential | Low-to-high magnitude | Light blue → dark blue for revenue scale |
| Diverging | Positive/negative from center | Red → white → green for performance |
| Categorical | Distinct unrelated categories | Blue, orange, green, purple for departments |
| Highlight | Emphasize one item | Gray everything, accent color on focus item |

## Typography in Infographics

| Role | Font Category | Size Range | Weight |
|------|-------------|-----------|--------|
| Title | Display or bold sans-serif | 36–60pt | Bold/Black |
| Section header | Sans-serif | 20–32pt | Semi-bold/Bold |
| Data label | Sans-serif | 12–18pt | Medium |
| Body text | Serif or sans-serif | 10–14pt | Regular |
| Caption/source | Sans-serif | 8–10pt | Light/Regular |

### Typography Rules for Infographics
- **Maximum 2 typefaces** — one for headings, one for body
- **Use tabular (monospace) figures** in charts — digits must align in columns
- **Left-align text blocks** — never center body text in infographics
- **Limit text to 30% of total space** — if text dominates, it's an article, not an infographic
- **High contrast only** — text must be readable at reduced sizes (sharing, mobile viewing)

## Layout Composition

### Common Layouts

| Layout | Width | Structure | Best For |
|--------|-------|-----------|----------|
| Vertical scroll | 800–1200px wide, variable height | Sections stacked vertically | Blog embedding, social sharing |
| Horizontal timeline | Variable width × 600–800px | Left-to-right progression | Timelines, processes |
| Poster/print | 18×24" to 24×36" | Grid-based sections | Physical display, conferences |
| Social media | 1080×1080 or 1080×1350 | Dense, limited sections | Instagram, LinkedIn |
| Interactive web | Responsive width | Scrollytelling sections | Website embedding |

### Composition Principles
- **Grid alignment** — align all elements to a consistent grid; misalignment looks amateur
- **Whitespace is content** — 30–40% of the infographic should be empty space
- **Group related items** — proximity implies relationship; use spacing to create visual groups
- **F-pattern flow** — readers scan left-to-right, then down; place key info along this path
- **Repetition** — consistent styling (icon style, chart colors, spacing) creates cohesion

## Production Quality Checklist

- ☐ All data verified against source
- ☐ Chart types appropriate for data relationships
- ☐ Axis labels, units, and legends present
- ☐ Color palette limited to ≤5 colors
- ☐ Colorblind-safe design verified
- ☐ Text-to-visual ratio <30% text
- ☐ Visual hierarchy clear (3 levels)
- ☐ Grid alignment consistent
- ☐ Source citations included
- ☐ No misleading scales or chart distortions
- ☐ Exported at correct resolution (300dpi print, 72dpi web)

## Best Practices

- **Start with the story, not the design** — know your narrative before opening a design tool
- **One key takeaway per infographic** — if you can't state the main point in one sentence, simplify
- **Data accuracy over aesthetics** — a beautiful chart with wrong numbers destroys credibility
- **Design for the smallest viewing size** — infographics get shared on mobile; ensure readability at small sizes
- **Use icons sparingly** — icons should clarify, not decorate; one style throughout
- **Show your sources** — data without attribution is opinion, not fact
