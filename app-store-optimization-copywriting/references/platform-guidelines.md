# Platform-Specific Guidelines

Detailed requirements for iOS and Android app stores.

---

## Apple App Store

### Indexed Elements
- App name (30 chars)
- Subtitle (30 chars)
- Keyword field (100 chars, hidden)
- In-App Purchase names

### Not Indexed
- Full description
- What's new
- Promotional text

### Promotional Text (170 chars)
- Appears above description
- Can change without app update
- Use for seasonal promotions, awards, features

### Best Practices
- Maximize keyword field efficiency
- Use subtitle for benefits
- Description for conversion only
- Consider IAP naming for keywords

---

## Google Play Store

### Indexed Elements
- App title (30 chars)
- Short description (80 chars)
- Full description (4,000 chars)
- Developer name

### No Hidden Keyword Field
- Must integrate keywords naturally
- Use variations and synonyms
- Reviews also indexed

### Best Practices
- Keywords 3-5 times in description
- Front-load first paragraph
- Use headings with keywords
- Natural, not stuffed

## Release Notes Best Practices

### Structure
```
🎉 NEW IN THIS UPDATE:
• [Feature 1]: [Brief explanation]
• [Feature 2]: [Brief explanation]

🛠️ IMPROVEMENTS:
• [Improvement 1]

🐛 BUG FIXES:
• [Fix 1]

Thanks for using [App Name]!
```

### Guidelines
- Be specific (not "bug fixes")
- Lead with exciting features
- Show you listen to feedback
- Keep it scannable
- Maintain brand voice
