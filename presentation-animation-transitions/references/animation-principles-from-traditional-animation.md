## Animation Principles from Traditional Animation

The 12 principles of animation, developed by Disney animators in the 1930s, remain foundational for creating natural, engaging motion. Several apply directly to presentation design:



### Timing and Pacing

**Timing** refers to how long an animation takes to complete. In presentations:

- **Quick animations (0.25-0.5 seconds)**: Efficient transitions, keeping pace brisk
- **Medium animations (0.5-1 second)**: Standard for most effects, natural feeling
- **Slow animations (1-2 seconds)**: Dramatic emphasis, contemplative moments
- **Very slow (2+ seconds)**: Special emphasis only; risks losing attention

**Best Practice**: Most presentation animations should fall between 0.3-0.75 seconds. Longer animations should be reserved for intentional dramatic effect.



### Easing and Acceleration

Natural motion includes acceleration and deceleration. Objects don't start and stop instantly—they ease in and out:

**Ease In**: Starts slowly, accelerates to full speed (object leaving)
**Ease Out**: Starts at full speed, decelerates to stop (object arriving)
**Ease In-Out**: Combines both (most natural for on-screen movement)
**Linear**: Constant speed (mechanical, less natural)

**Application**: Always apply easing to animations. Linear motion feels robotic and unpolished. Most presentation software defaults to eased animations.



### Anticipation

Anticipation prepares the audience for an action. In presentations:

- A slight zoom back before zooming forward
- A brief pause before revealing key information
- Dimming other elements before highlighting one

This principle builds suspense and draws attention to upcoming content.



### Follow-Through

After main action completes, elements may continue moving slightly:

- Text that overshoots then settles into position
- Bouncing effect after an element lands
- Subtle oscillation after rotation stops

Follow-through adds realism but should be used sparingly in professional contexts.



### Staging

Staging presents ideas clearly, one at a time. In presentations:

- Reveal bullet points sequentially, not all at once
- Build charts element by element
- Isolate key messages before introducing supporting details

Effective staging ensures the audience knows where to look and what to focus on.



### Secondary Action

Supporting animations that complement the main action:

- Background dimming while highlighting foreground elements
- Supporting icons animating alongside main content
- Decorative elements responding to primary animations

Secondary actions should support, not compete with, primary content.
