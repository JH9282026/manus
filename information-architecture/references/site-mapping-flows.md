# Site Mapping and User Flows

## Overview

Site maps and user flows visualize information architecture and user journeys. These artifacts communicate structure to stakeholders and guide design decisions.

---

## Site Maps

**Purpose**: Visual representation of site structure. Shows hierarchy and relationships between pages.

**Components**:
- Pages/Screens: Represented as boxes or cards
- Hierarchy: Parent-child relationships shown through indentation or connecting lines
- Page types: Different shapes or colors for different types (landing page, article, product, etc.)
- Annotations: Notes about functionality, content, or requirements

**Levels of Detail**: 
- High-level: Main sections only. For stakeholder communication.
- Detailed: Every page and subpage. For development planning.

**Format**: Visual diagram (Figma, Lucidchart, Miro) or hierarchical list (spreadsheet, outline).

**Best Practices**: Keep it simple, use consistent notation, update as structure changes, share with team.

---

## User Flows

**Purpose**: Visualize paths users take to complete tasks. Shows decision points, actions, and outcomes.

**Components**:
- Entry points: Where user starts (homepage, email link, search result)
- Actions: What user does (click, type, select)
- Decision points: Branching based on user choice or system state
- Pages/Screens: What user sees at each step
- End points: Task completion or exit

**Notation**:
- Rectangles: Pages or screens
- Diamonds: Decision points
- Arrows: Flow direction
- Circles: Start and end points

**Example - Password Reset Flow**:
1. User clicks 'Forgot Password'
2. System shows email input
3. User enters email
4. System sends reset link
5. User clicks link in email
6. System shows password reset form
7. User enters new password
8. System confirms and logs user in

---

## Task Flows vs User Flows

**Task Flows**: Single path to complete specific task. Linear, focused. Example: 'Complete checkout'.

**User Flows**: Multiple paths and decision points. Shows all possible routes. Example: 'Shopping experience' including browsing, searching, comparing, adding to cart, checkout.

**When to Use Task Flows**: Designing specific features, optimizing conversion funnels, documenting requirements.

**When to Use User Flows**: Understanding overall experience, identifying pain points, mapping complex journeys.

---

## Flow Diagrams Best Practices

**Start with Research**: Base flows on actual user behavior, not assumptions. Use analytics and user testing.

**Focus on Key Paths**: Don't try to show every possible path. Focus on primary user journeys.

**Show Happy Path and Errors**: Include both successful completion and error states.

**Annotate Decisions**: Explain branching logic. 'If user is logged in' vs 'If user is guest'.

**Keep It Readable**: Don't cram too much into one diagram. Create multiple flows if needed.

**Use Consistent Notation**: Stick to standard flowchart symbols or create a legend.

**Validate with Users**: Walk through flows with actual users. Do they match mental models?

## Tools for Site Maps and Flows

**Figma/Sketch/Adobe XD**: Design tools with flowchart capabilities. Good for visual polish.

**Lucidchart**: Purpose-built for diagrams. Extensive shape libraries and templates.

**Miro/Mural**: Collaborative whiteboarding. Great for workshops and team collaboration.

**Whimsical**: Simple, fast flowcharting. Clean aesthetic, easy to learn.

**Draw.io**: Free, open-source diagramming. Desktop or web-based.

**OmniGraffle**: Mac-only, powerful diagramming. Professional output.

**Choosing a Tool**: Consider collaboration needs, integration with design tools, learning curve, and cost.

---

## Communicating IA to Stakeholders

**Executive Summary**: High-level overview of structure and rationale. Why this approach?

**Visual Site Map**: Show overall structure at a glance. Use color coding for different sections.

**Key User Flows**: Demonstrate how users will accomplish primary tasks. Walk through step-by-step.

**Comparison**: If proposing changes, show before and after. Highlight improvements.

**Research Support**: Reference user research, card sorting results, analytics. Ground decisions in data.

**Prototype**: Interactive prototype brings IA to life. More tangible than diagrams.

**Presentation Tips**: Tell a story, focus on user benefits, anticipate questions, be open to feedback.
